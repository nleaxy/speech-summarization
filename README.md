# AI Speech Summarizer

Full-Stack веб-приложение для транскрибации и суммаризации аудиофайлов, упакованное в Docker для максимально простого и надежного запуска.

## Стек технологий

*   **Бэкенд:** Python, Flask, Gunicorn
*   **Транскрибация:** OpenAI Whisper (локальная модель)
*   **Суммаризация:** Внешнее API (OpenRouter.ai с моделью Mistral)
*   **Фронтенд:** React, TypeScript, Vite, Tailwind CSS
*   **Окружение:** Docker, Docker Compose

---

## ✅ Инструкция по Запуску

Этот способ является **рекомендуемым**, так как он гарантирует, что проект запустится на любой операционной системе (Windows, macOS, Linux) без ручной установки Python, FFMPEG и других зависимостей.

### 1. Предварительные требования

*   **[Git](https://git-scm.com/downloads)** — для скачивания проекта.
*   **[Docker Desktop](https://www.docker.com/products/docker-desktop/)** — для запуска приложения. **Убедитесь, что Docker Desktop запущен** (иконка кита в трее/меню-баре должна быть статичной).

### 2. Запуск

1.  **Клонируйте репозиторий:**
    Откройте терминал (командную строку) и выполните команду:
    ```bash
    git clone https://github.com/nleaxy/speech-summarization.git
    cd speech-summarization
    ```
    *(Замените URL на свой, если он отличается)*

2.  **Запустите скрипт для вашей операционной системы:**

    *   #### ➡️ Для Windows:
        Просто найдите в папке проекта файл `start.bat` и **дважды кликните по нему**.

    *   #### ➡️ Для macOS / Linux:
        Сначала нужно сделать скрипт исполняемым. Откройте терминал в папке проекта и выполните **один раз**:
        ```bash
        chmod +x start.sh
        ```
        Затем запустите его:
        ```bash
        ./start.sh
        ```

3.  **Следуйте инструкциям в терминале.**
    При первом запуске скрипт попросит вас получить и вставить API-ключ от сервиса `OpenRouter.ai`. Это нужно сделать только один раз. Скрипт предоставит прямую ссылку на страницу для создания ключа.

4.  **Дождитесь окончания сборки.**
    Docker начнет собирать приложение. **Первый запуск может занять 5-15 минут**, так как будут скачиваться все необходимые компоненты и ML-модели. Последующие запуски будут почти мгновенными.

5.  **Откройте приложение в браузере.**
    Когда в терминале появятся логи от веб-сервера, откройте браузер и перейдите по адресу:
    [**http://localhost:5000**](http://localhost:5000)

### Известные проблемы

*   **Не запускайте обработку нескольких файлов на нескольких страницах сразу.**
    Пока что приложение способно обрабатывать лишь один файл за раз.

*   **Не обновляйте и не закрывайте страницу во время обработки файла.**
    Файл продолжит обрабатываться на фоне, но результат не выведется на странице приложения.


###  troubleshoot/FAQ

*   **Ошибка `docker` или `docker-compose` not found:**
    Убедитесь, что Docker Desktop установлен и запущен. Попробуйте перезапустить терминал.

*   **Я ввел неправильный API-ключ, что делать?**
    Просто удалите файл `backend/.env`, который создал скрипт, и запустите `start.bat` или `start.sh` заново. Он снова попросит вас ввести ключ.

*   **Как остановить приложение?**
    Вернитесь в терминал, в котором запущено приложение, и нажмите `Ctrl + C`.

*   **Как запустить приложение снова (не в первый раз)?**
    Просто повторите шаг 2. Сборка будет пропущена (если код не менялся), и запуск будет очень быстрым.
