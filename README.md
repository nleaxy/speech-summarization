# AI Speech Summarizer

Full-Stack веб-приложение для транскрибации и суммаризации аудиофайлов, упакованное в Docker для максимально простого и надежного запуска.

## Стек технологий

*   **Бэкенд:** Python, Flask, Gunicorn
*   **Транскрибация:** OpenAI Whisper (локальная модель `medium`)
*   **Суммаризация:** Внешнее API (OpenRouter.ai)
*   **Фронтенд:** React, TypeScript, Vite, Tailwind CSS
*   **Окружение и развертывание:** Docker, Docker Compose

---

## ✅ Инструкция по Запуску

Этот способ является **рекомендуемым**, так как он гарантирует, что проект запустится на любой операционной системе (Windows, macOS, Linux) без ручной установки Python, FFMPEG и других зависимостей.

### 1. Предварительные требования

*   **[Git](https://git-scm.com/downloads)** — для скачивания проекта.
*   **[Docker Desktop](https://www.docker.com/products/docker-desktop/)** — для запуска приложения. **Убедитесь, что Docker Desktop запущен** (иконка кита в трее/меню-баре должна быть статичной).

### 2. Запуск

1.  **Клонируйте репозиторий:**
    Откройте терминал (командную строку) и выполните команды:
    ```bash
    git clone https://github.com/nleaxy/speech-summarization.git
    cd speech-summarization
    ```
    *(Убедитесь, что URL репозитория правильный)*

2.  **Запустите скрипт для вашей операционной системы:**

    ---
    #### ➡️ **Способ для Windows**

    **Простой способ (через Проводник):**
    1.  Откройте папку с проектом в Проводнике Windows.
    2.  Кликните правой кнопкой мыши по файлу `start.ps1`.
    3.  В контекстном меню выберите **"Выполнить с помощью PowerShell"** ("Run with PowerShell").

    **Способ для терминала (рекомендуемый):**
    1.  Откройте терминал (PowerShell или терминал в VS Code) в корневой папке проекта.
    2.  Выполните команду:
        ```powershell
        .\start.ps1
        ```

    *Если PowerShell выдаст ошибку безопасности о выполнении скриптов, выполните один раз команду `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`, а затем снова запустите `.\start.ps1`.*

    ---
    #### ➡️ **Способ для macOS / Linux**

    1.  Откройте терминал в корневой папке проекта.
    2.  Сначала сделайте скрипт исполняемым (это нужно сделать **только один раз**):
        ```bash
        chmod +x start.sh
        ```
    3.  Теперь запустите его:
        ```bash
        ./start.sh
        ```
    ---

3.  **Следуйте инструкциям в терминале.**
    При первом запуске скрипт попросит вас получить и вставить API-ключ от сервиса `OpenRouter.ai`.

4.  **Дождитесь окончания сборки и загрузки моделей.**
    Docker начнет собирать приложение. **Первый запуск может занять 10-20 минут**, так как будут скачиваться все компоненты и большая ML-модель Whisper (~1.5 ГБ).

5.  **Откройте приложение в браузере.**
    Когда в терминале появятся логи от веб-сервера `gunicorn`, откройте браузер и перейдите по адресу:
    [**http://localhost:5000**](http://localhost:5000)

### ⚠️ Известные Ограничения (MVP)

*   **Обработка только одного файла за раз:** Из-за синхронной архитектуры бэкенда, сервис может корректно обрабатывать только один запрос на транскрибацию в один момент времени. Не запускайте обработку нескольких файлов одновременно.

*   **Потеря состояния при перезагрузке:** Если вы обновите или закроете страницу во время обработки, связь с выполняемой задачей будет потеряна. Результат не отобразится, хотя сервер может завершить обработку в фоновом режиме.

###  troubleshoot/FAQ

*   **Ошибка `docker` или `docker-compose` not found:**
    Убедитесь, что Docker Desktop установлен и запущен. Попробуйте перезапустить терминал.

*   **Я ввел неправильный API-ключ, что делать?**
    Просто удалите файл `backend/.env`, который создал скрипт, и запустите **`start.ps1`** (для Windows) или **`./start.sh`** (для macOS/Linux) заново.

*   **Как остановить приложение?**
    Вернитесь в терминал, в котором запущено приложение, и нажмите `Ctrl + C`.

*   **Как запустить приложение снова (не в первый раз)?**
    Просто повторите шаг 2. Сборка будет пропущена, модели загрузятся из кэша, и запуск будет очень быстрым.
